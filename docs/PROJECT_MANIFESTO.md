# 🚀 Project Manifesto: The Art of "Zero-Loading"

## 1. 🎯 The Core Problem: 단 하나의 질문
> **"10만 명이 동시에 '로그인 버튼'을 누를 때, 1초 내에 서버가 응답할 수 있는가?"**

이 프로젝트는 단순한 '인증 서버'를 만드는 것이 아닙니다. 티켓팅이나 선착순 이벤트에서 발생하는 **'로그인 무한 로딩'**이라는 절망적인 사용자 경험을 기술적으로 완전히 박멸하기 위한 도전입니다. 우리의 모든 코드는 **"대용량 트래픽에서의 무중단 인증"**이라는 하나의 목표를 위해 정렬됩니다.

---

## 🏗️ The Problem-Solving Map: 우리의 전투와 무기

우리는 기술을 유행으로 선택하지 않습니다. 각 단계는 우리가 마주할 처절한 병목 현상을 타파하기 위한 최선의 '무기'였습니다.

### Challenge 01: 중앙 집중형 저장소의 I/O 병목 (The Session Limit)
*   **문제 (Pain Point)**: 서버를 10대로 늘려도 모든 서버가 세션 확인을 위해 하나의 Redis(SPOF)에 매달리면서 CPU 90%를 상회하는 병목 발생. "서버 확장의 Scalability의 물리적 한계".
*   **무기 (Weapon)**: **Stateless JWT**. 각 서버가 스스로 토큰의 유효성을 검증하게 하여 중앙 저장소 조회를 0회로 수렴시킴. 서버 수에 비례하는 무한한 확장성 확보.

### Challenge 02: 보안과 성능의 치명적 트레이드오프 (The Security Gap)
*   **문제 (Pain Point)**: Stateless 구조의 효율성은 얻었으나, 한번 발행된 토큰을 즉시 무효화할 수 없는 보안적 맹점이 존재. "성능을 위해 보안을 포기할 것인가?"
*   **무기 (Weapon)**: **RTR (Refresh Token Rotation)**. 매번 토큰을 갈아 치우는 '1회용 토큰' 전략을 통해, 최소한의 I/O로 Stateless의 효율성과 세션 수준의 보안성을 동시에 획득.

### Challenge 03: 코드의 불투명성과 런타임 불안정성 (The Safety Net)
*   **문제 (Pain Point)**: 대규모 트래픽 로직을 다룰 때 Java의 상황별 코드와 Null 위험은 장애의 주범임. 극한 상황에서의 코드 한 줄의 실수는 시스템 전체의 마비를 가져옴.
*   **무기 (Weapon)**: **Kotlin & Functional Idioms**. Null-Safety를 컴파일 타임에 강제하고, 선언적인 코드로 '실제 지능 로직'만 명확히 드러나게 함으로써 안정성을 극대화.

### Challenge 04: 매크로와 비정상 트래픽의 파동 (The Traffic Flood)
*   **문제 (Pain Point)**: 정상 사용자의 10배에 달하는 매크로 공격은 인증 시스템을 마비시킴. 아무리 성능이 좋아도 비정상 트래픽을 거르지 못하면 시스템은 무너짐.
*   **무기 (Weapon)**: **Rate Limiting & Distributed Lock**. 서비스 진입층(Filter Layer)에서 비정상 요청을 즉각 차단하여 엔진 부하를 보호하는 '방호벽 구축.

## 📈 검증의 기준 (The Success Metrics)

우리는 '작동한다'에 만족하지 않습니다. 다음 가상 시나리오를 통과해야만 '성공'으로 간주합니다.

### [Scenario: The 10:00 AM Spike]
1.  **Latency**: 99.9%의 사용자가 **100ms 이내**에 토큰을 발급받아야 함.
2.  **Linear Scalability**: 서버 인스턴스 2배 증설 시 처리량(TPS)이 **1.8배 이상** 선형적으로 증가해야 함.
3.  **Resilience**: 인증 DB가 일시적으로 지연되더라도, 이미 로그인된 사용자의 서비스 이용은 **지연 시간 0ms**로 유지되어야 함.

## 💡 Conclusion: 기술보다 '이유'
우리가 기술을 선택하는 이유는 그것이 최신이기 때문이 아닙니다. **그것만이 우리의 문제를 풀 수 있기 때문**입니다. 이 프로젝트는 **"기술적 수단이 목표(로딩 0초)를 어떻게 현실로 만드는가"**를 증명하는 하나의 거대한 실험입니다.
